Colors in RGB

A computer screen is made of microscopic red, blue, and green lights. The computer sets each light's brightness with a number between 0 (or 000) and 255.

A pixel is a tiny square that has one of each of these lights. A bright purple pixel has an RGB code of 255 000 255 which means the pixel's red light is on, green is off, and blue is on.

170 000 170 makes a darker purple: the red and blue lights are a little dim, and green is still off.

Colors in Hex

We use the decimal system to write numbers. Computers can save memory by using the hexadecimal system, or hex. It uses letters in place of some numbers: A for 10, B for 11, up to F for 15.

Mixing Colors

Knowing how hex works makes it easier to find the perfect color for your website or app.

If you want yellow, but ffff00 is too yellow, you can giving it 1 more brightness of red with ffff01. You can also make it 1 less brightness of green with fffe00, or 16 less brightness of green with ffef00.

In CSS, you pick a font using the font-family property. For example:

font-family: "Times New Roman", Times, serif;
font-family: cursive;
font-family: "Courier New", monospace;

If a font in font-family isn't installed on your computer, your computer tries the next one. If instead of a font it's a family, like cursive, your computer uses any font in that family.

A font name that contains a 1 must be inside "quotes".

Stylesheets

All of a website's CSS is usually kept in a file called a stylesheet:

1..  p, code{
2..    color: #000000;
3..    font-size: 18px;
4..  }
5..  p{
6..    font-family: sans-serif;
7..  }
8..  code{
9..    font-family: monospace;
10.  }

Each of the chunks in {braces} is a rule. The p rule changes all the paragraphs on the website. The p, code rule changes all the paragraphs and code snippets. Rules add on to each other.

This stylesheet changes the fonts of all the code snippets to be 1.

  1. black, 18 pixels tall, and monospaced
  2. black, 18 pixels tall, and sans-serif
  3. monospaced
  4. sans-serif

Selectors

Every CSS rule starts with a selector. It tells the computer which parts of a website it should select to use with that CSS rule.

p, code, img{
  width: 400px;
}

The selector here is p, code, img. It means, Every paragraph, code snippet, and image should follow this rule.

p code img{
  width: 400px;
}

Without commas the meaning is totally different. It means, Every image that's in a code snippet that's in a paragraph.

4 selects every button in every form.

  1. button, form
  2. form, button
  3. button form
  4. form button
  1. Every button in every paragraph in every form. form p button
  2. Every button, every paragraph, and every form. button, p, form
  3. Every button and every form. button, form

Cascading Stylesheets

...is what CSS stands for. Cascading means three things:

  1. If a website has two stylesheets, it uses both of them.
  2. If a website has two CSS rules for paragraphs, it uses both rules.
  3. ...but if there are two font-family declarations for paragraphs, the website will use the second declaration. 
1..  p{
2..    background-color: #000000;
3..    color: #FFFF00;
4..  }
5..  code, p{
6..    background-color: #00FF00;
7..  }
8..  code{
9..    background-color: #000000;
10.    background-color: #0000FF;
11.  }
